# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-07
**Phases:** 1-8
**Total Plans:** 8

## Overview

Initial MVP of LeClaw — an OpenClaw orchestration center for managing hierarchical AI agents across multiple companies. Delivers CLI, REST API, Web UI, and E2E test infrastructure.

## Phases

### Phase 1: Foundation - CLI Init

**Goal**: User can initialize LeClaw project and configure settings via CLI
**Depends on**: Nothing (first phase)
**Plans**: 1 plan

Plans:

- [x] Phase 1: Foundation + CLI Init

**Details:**
Project scaffolding with `leclaw init` command, `leclaw config` command for managing openclaw.dir, gatewayUrl, gatewayToken, and server.port settings. Interactive TUI prompts for configuration. Health check and doctor commands.

---

### Phase 2: Data Layer - Entity Models + DB

**Goal**: All core entities persist correctly in embedded PostgreSQL with Drizzle ORM
**Depends on**: Phase 1
**Plans**: 1 plan

Plans:

- [x] Phase 2: Data Layer Entity Models + DB

**Details:**
8 entity schemas: Company, Department, Agent, Issue, IssueComment, Goal, Project, Approval. Embedded PostgreSQL via embedded-postgres. Drizzle ORM with migrations. UUID v4 IDs, text enums, JSONB arrays for flexible collections, companyId-based indexing for tenant isolation.

---

### Phase 3: OpenClaw Integration

**Goal**: LeClaw discovers OpenClaw agents and supports agent onboarding
**Depends on**: Phase 2
**Plans**: 1 plan

Plans:

- [x] Phase 3: OpenClaw Integration

**Details:**
OpenClaw agent discovery via `openclaw.json` scanning. `leclaw agents list` command. `leclaw agent onboard` with 5-rule validation. API key generation and storage. Gateway client for status queries.

---

### Phase 4: REST API + SSE

**Goal**: All entities exposed via REST API; SSE for real-time Web UI updates
**Depends on**: Phase 3
**Plans**: 1 plan

Plans:

- [x] Phase 4: REST API + SSE

**Details:**
Express-based REST API with CRUD for all entities. SSE endpoint with 30-second heartbeat. Event broadcasting for real-time updates. Nested routes under `/api/companies/:companyId/` for tenant-scoped resources.

## Milestone Summary

**Decimal Phases:**

(None)

**Key Decisions:**

- Independent process connecting to OpenClaw (Rationale: Does not modify OpenClaw, maintains flexibility)
- React Web UI (Rationale: User specified technology stack)
- SSE for real-time updates (Rationale: Reference openclaw-control-center implementation)
- Embedded PostgreSQL (Rationale: Reference paperclip design)
- Fixed two-level hierarchy (Company/Department) (Rationale: Simplifies complexity for MVP)
- Hybrid collaboration mode (Rationale: Supports assignment + acceptance + parallel collaboration)
- Zero Auth design (Rationale: Single user managing multiple companies)
- Express over Hono (Rationale: Existing codebase used Express, maintained compatibility)

**Issues Resolved:**

- embedded-postgres beta version compatibility issues
- Drizzle ORM type definition errors (skipLibCheck workaround)
- @clack/prompts broken default import pattern
- React Router nested route active state highlighting

**Issues Deferred:**

- Issue auto-assignment based on routing rules (v2)
- CEO agent decompose goals to Managers (v2)
- Manager agents coordinate Staff for parallel execution (v2)
- Agent completion triggers automatic Issue update (v2)
- Full audit log viewer UI (v2)
- Audit log retention/rotation policy (v2)
- bcrypt hashing for API keys (v2)

**Technical Debt Incurred:**

- API keys stored with plaintext secrets (should use bcrypt in production)
- No database migration automation for embedded-postgres
