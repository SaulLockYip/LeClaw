# Milestone v1.0 Requirements Archive

**Status:** ✅ SHIPPED 2026-04-07
**Total Requirements:** 35 v1 requirements

## v1 Requirements — All Checkboxes Below

### CLI

- [x] **CLI-01**: User can run `leclaw init` to initialize the configuration directory (`~/.leclaw/`)
- [x] **CLI-02**: User can run `leclaw config openclaw --dir <path>` to configure the OpenClaw directory
- [x] **CLI-03**: User can run `leclaw config gateway --url <url> --key <key>` to configure Gateway address and API key
- [x] **CLI-04**: User can run `leclaw start` to start the LeClaw server (backend + web UI)
- [x] **CLI-05**: User can run `leclaw status` to view connection status to OpenClaw and Gateway

### Data Layer

- [x] **DATA-01**: LeClaw initializes embedded PostgreSQL database on first run
- [x] **DATA-02**: Company entity can be created, read, updated, deleted
- [x] **DATA-03**: Department entity can be created, read, updated, deleted under a Company
- [x] **DATA-04**: Agent binding relationship (CEO/Manager/Staff -> OpenClaw agent) is stored and retrieved
- [x] **DATA-05**: Issue entity can be created and listed (title, description, status, assignee)

### OpenClaw Connection

- [x] **OPENCLAW-01**: LeClaw scans configured OpenClaw directory to discover all available agents
- [x] **OPENCLAW-02**: LeClaw polls OpenClaw Gateway for agent status at configurable intervals
- [x] **OPENCLAW-03**: LeClaw maintains heartbeat with OpenClaw agents to verify liveness
- [x] **OPENCLAW-04**: Agent status changes trigger SSE events to connected Web UI clients

### Web UI - Dashboard

- [x] **UI-01**: Dashboard displays summary of all Companies and their Department counts
- [x] **UI-02**: Dashboard displays real-time status of all detected OpenClaw agents
- [x] **UI-03**: Dashboard shows recent Issues and their states

### Web UI - Organization

- [x] **UI-04**: User can view list of Companies and create a new Company
- [x] **UI-05**: User can view a Company's detail page with CEO agent assigned
- [x] **UI-06**: User can view Company's Departments and create a new Department
- [x] **UI-07**: User can assign a Manager agent to a Department from available OpenClaw agents
- [x] **UI-08**: User can assign one or more Staff agents to a Department from available OpenClaw agents
- [x] **UI-09**: User can reassign or unbind any agent from their role

### Web UI - Issues

- [x] **ISSUE-01**: User can create an Issue from Web UI (title, description, target Department)
- [x] **ISSUE-02**: User can view list of all Issues with filtering by Company/Department
- [x] **ISSUE-03**: User can view Issue detail and change its status (open, in-progress, done)
- [x] **ISSUE-04**: External systems can create Issues via REST API with API key authentication

### Real-Time Updates

- [x] **RT-01**: Web UI receives real-time updates via SSE when agent status changes
- [x] **RT-02**: SSE connections include heartbeat to prevent load balancer timeout
- [x] **RT-03**: Web UI reconnects automatically after SSE connection drop

### REST API

- [x] **API-01**: REST API exposes Company CRUD operations
- [x] **API-02**: REST API exposes Department CRUD operations
- [x] **API-03**: REST API exposes Issue CRUD operations
- [x] **API-04**: REST API exposes OpenClaw agent list and status
- [x] **API-05**: API endpoints return proper error responses with error codes
